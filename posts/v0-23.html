Vercel, PWA, Accessibility und Passwortschutz

20. Mai 2026 · v0.23

Infrastruktur-Release: die Edition zieht von GitHub Pages auf Vercel um, wird zur installierbaren App und bekommt Passwortschutz für die Closed Alpha.

Migration auf Vercel

Das Deployment läuft jetzt über Vercel statt GitHub Pages. Vorteile: Edge-Middleware für Passwortschutz, saubere URLs ohne .html, schnelleres CDN, und Preview-Deployments bei jedem Push.

Passwortschutz (Closed Alpha)

Die Edition ist jetzt durch HTTP Basic Auth geschützt. Eine Vercel Edge Middleware prüft Benutzername und Passwort, bevor irgendein Inhalt ausgeliefert wird. So bleibt die Alpha kontrolliert, bis die Textqualität stimmt.

Progressive Web App

  • Service Worker cached die Grundstruktur (HTML, CSS, JS) für schnellere Ladezeiten
  • Stale-while-revalidate für Assets, Network-first für Seiteninhalte
  • Web-Manifest — die Edition kann auf Mobilgeräten und Desktop als App installiert werden
  • Offline-Fallback: bei Netzwerkausfall wird die 404-Seite angezeigt
  • SVG-Favicon

404-Seite

Eigene Fehlerseite im Design der Edition mit Navigation zurück zur Sammlung oder Volltextsuche. Wird auch vom Service Worker als Offline-Fallback verwendet.

Accessibility

  • aria-label, aria-live, aria-expanded, aria-current durchgehend ergänzt
  • Toolbar-Eingaben verwenden jetzt <label> statt <span>
  • Skip-Links („Zum Text springen“, „Zur Suche springen“)
  • Mobile Sidebar: Backdrop-Overlay sperrt den Body-Scroll
  • focus-visible-Outline mit Gold-Akzent

SEO

  • OpenGraph- und Twitter-Meta-Tags auf jeder Seite
  • Kanonische URLs (<link rel="canonical">)
  • JSON-LD (schema.org WebSite) auf der Startseite
  • <meta name="description"> auf jeder Seite
